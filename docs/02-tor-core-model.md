# TOR Core Model

This document defines the canonical structure of a TOR document.

---

## Core Sections

A compliant TOR document should contain the following sections.

| Section                   | Description               | Required |
| ------------------------- | ------------------------- | -------- |
| Background                | Context and rationale     | Yes      |
| Objectives                | Project goals             | Yes      |
| Scope of Work             | Activities required       | Yes      |
| Deliverables              | Expected outputs          | Yes      |
| Timeline                  | Schedule or milestones    | Yes      |
| Acceptance Criteria       | Conditions for acceptance | Yes      |
| Assumptions & Constraints | Known limitations         | Optional |
| Dependencies              | External dependencies     | Optional |
| Budget                    | Budget information        | Optional |
| Vendor Qualification      | Required qualifications   | Optional |
| Evaluation Criteria       | Proposal evaluation rules | Optional |
